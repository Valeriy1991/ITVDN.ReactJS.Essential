#### 1 урок. Redux intro

Приведу краткое содержание.
* Фичи Redux:
    * Hot Module Replacement
    * Time traveling
    * Single Store
    * Immutable Store
    * Простой API
    * Middlewares
* Когда нужен Redux:
    * Гонка состояний
    * Сложный поток данных (когда компоненты должны общаться друг с другом)
    * Одни и те же данные в разных местах
    * Множество действий (экшенов)
* 3 принципа Redux:
    * Store - read only
    * Actions
    * Reducers
